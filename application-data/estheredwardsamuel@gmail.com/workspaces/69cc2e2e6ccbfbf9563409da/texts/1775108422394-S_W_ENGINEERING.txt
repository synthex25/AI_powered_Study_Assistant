🧩 What is OOSE
Software development using object-oriented principles
Focus on objects, classes, and interactions
Combines OOP + Software Engineering practices
🧠 Key Concepts
Object → real-world entity (has data + behavior)
Class → blueprint of objects
Encapsulation → hide data
Inheritance → reuse code
Polymorphism → one interface, many forms
Abstraction → show only essentials
🔄 OOSE Process
Requirement Analysis
System Design
Implementation
Testing
Maintenance
📊 UML in OOSE

Use Case Diagram

Classes
Attributes

🔁 Sequence Diagram
Shows flow of messages

Easy to maintain
